This GINA has the following features:

  • Compatible with Windows 2000 up to Windows 2003 Server.
  • Graphical configuration facility!
  • Fully configurable via registry (and therefore also remotely).
  • Configuration settings take place immediately as the GINA watches its registry key for configuration changes.
  • Showing an agreement for the usage of the machine as HTML.
  • Forcing the user to wait a certain time before he is allowed to proceed with logon. I.e. forcing him to read the agreement.
  • Optionally showing news (HTML).
  • Special maintenance mode which denies logon to ordinary non-admin users.
  • Automatic guest logon. Given a username and password for the "guest" account, the GINA will only allow admins to override this automatic logon and will else stay in logged-on mode unless being shutdown.
  • Suppressing "Lock workstation", "Change password" and "Task manager" separately and optionally for non-admin users. Admins will always see all these items.
  • Easy credential configuration via registry. I.e. setting the username and password for "guest" can be done via registry. The GINA will take them and put them into the password stash of the system, deleting the "volatile" registry values.
  • Optionally: Information about the logged on user (time of logon, name, domain, guest-only/maintenance/admin status) via very simple server at a configurable port!

Brief version history:

The first version of this software was written in Borland Delphi 4. Due to implementation problems I decided to rewrite the whole piece of software in C++, namely Visual C++ 6.0.
This is the rewrite and it comes with more features than the original version. E.g. it is now possible to configure the GINA directly on the system running it. This can be achieved by replacing the "Logoff" button in the Windows security dialog with the configuration for admin users.
For more information, check out the text files in the distribution.
