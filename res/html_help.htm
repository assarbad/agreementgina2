AgreementGINA2: configuration help

Wanna back to the about dialog?

This is a very short description of the configuration options in the dialog which led you here.

  1. Settings for the agreement dialog
    • Agreement URL lets you set the (HTML) file to show as agreement.Alternatively you may give HTML content here. Note, that the first character must be "<" and it must not contain linebreaks.
    • Show timer must be checked to show the timer countdown if duration is not zero. If the duration is zero and this is checked, the agreement dialog will not be shown.
    • The timer duration (the edit field) can be zero (see "Show timer" above) or up to 3600 seconds.
    Once more: checking "Show timer" and setting the "timer duration" to zero will completely turn off the agreement dialog!
    Note: The URL field will show with blue background if it is being recognized as a URL, but with white background if it is recognized as HTML content!
  2. News message and maintenance mode
    • News URL sets the HTML file to be shown as news (if activated). Alternatively you may give HTML content here. Note, that the first character must be "<" and it must not contain linebreaks.
    • Maintenan. URL sets the HTML file to be shown as maintenance notice (if activated). Alternatively you may give HTML content here. Note, that the first character must be "<" and it must not contain linebreaks.
    • Maintenance mode active activates or deactivates the maintenance mode.
    • News message active activates or deactivates showing of the news message.
    Note: The URL fields will show with blue background if it is being recognized as a URL, but with white background if it is recognized as HTML content!
  3. Restrictions for non-administrative users
    These options handle whether the respective buttons are active or not in the security dialog (the one you see when being logged on and pressing Ctrl+Alt+Del).
  4. Caption of controls (agreement dialog)
    These options set the texts shown for the Yes (proceed) button, the No button and the checkbox which needs to be checked for the Yes button to become active.
  5. General settings
    • Guest user sets the name of the account to be used for "guest only" mode.
    • Guest password and confirm the password for the guest user.
    • Allow only guest logon activates/deactivates "guest only" mode.
    • Admin skips agreement and news will not show the admin any of the dialogs (agreement, news) when activated.
    • Listen on port sets whether to listen on a TCP port and on which. Check before with TELNET whether the respective port is free. Connecting to this port a HTML string will be returned containing data about the currently logged on user.
    Important note: if the user/password fields show with red background color, they will not be saved (i.e. passwords do not match). If the passwords match these fields will appear green and the settings be written.
